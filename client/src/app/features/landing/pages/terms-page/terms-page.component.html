Terms of use

These terms describe the intended usage of the current MVP: authenticated collaboration, project membership, task tracking, and communication inside a controlled workspace.

Access

Users should access only the projects and actions allowed by their role. Project roles and team roles determine what can be viewed or changed.

Content responsibility

Users are responsible for the tasks, comments, and files they create inside the workspace, especially in shared project contexts.
